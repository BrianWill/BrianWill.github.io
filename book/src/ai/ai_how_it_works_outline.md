# AI Explainer: Technical Concepts and Terminology

## I. Fundamental Building Blocks
- **Tensor**
    - Multi-dimensional data arrays; the primary mathematical primitive for deep learning.
    - Generalization of scalars (0D), vectors (1D), and matrices (2D).
- **Perceptron**
    - The historical predecessor to modern neural network nodes.
    - A linear classifier that maps inputs to a binary output via a weight-sum and activation function.
- **Layers**
    - Discrete stages of processing within a model.
    - Input, hidden, and output layers transform data representations hierarchically.
- **Neural Network**
    - A computational system of interconnected layers (neurons) that learns patterns by adjusting internal weights.

## II. Training and Optimization
- **Model**
    - The static artifact resulting from training; a specific configuration of weights and architecture capable of inference.
- **Gradient Descent**
    - The primary optimization algorithm used to minimize the loss function.
    - Iteratively adjusts weights in the direction of steepest descent of the error gradient.
- **Training Phases**
    - **Pre-training**: Training on vast, unlabelled datasets (e.g., Common Crawl) to create a **Base Model** with general world knowledge and language capabilities.
    - **Post-training**: Refining the base model for specific behaviors.
        - **Supervised Fine-tuning (SFT)**: Learning from curated instruction-response pairs.
        - **Reinforcement Learning (RL)**: 
            - Technical definition: Optimizing a policy to maximize a reward signal (e.g., RLHF - Reinforcement Learning from Human Feedback).
            - Popular definition: Any iterative improvement based on "correcting" model outputs.
- **Distillation**
    - Training a smaller "student" model to mimic the output distribution of a larger "teacher" model to reduce computational overhead.
- **Open Models**
    - Models whose weights are publicly downloadable (e.g., Llama, Mistral), distinguishing them from "Closed" API-only models (e.g., GPT-4, Claude 3).

## III. Architecture: The Transformer Revolution
- **Transformer**
    - The dominant architecture for sequence processing (introduced in "Attention Is All You Need").
    - Replaced sequential processing (RNNs/LSTMs) with parallelizable attention mechanisms.
- **Attention (Self-Attention)**
    - A mechanism allowing the model to weight the relevance of different parts of the input sequence relative to each other.
- **Diffusion**
    - The primary architecture for image generation (e.g., Stable Diffusion) before the recent shift toward using Transformers for vision-language tasks.
- **Large Language Model (LLM)**
    - Transformers scaled to billions of parameters and trained on massive text corpora.
- **Vision-Language Model (VLM)**
    - Models capable of processing both image and text inputs natively (e.g., GPT-4o, Gemini 1.5 Pro).

## IV. Tokens and Inference
- **Tokens**
    - The atomic units of text processing; models do not see "words" but integers representing sub-word segments (e.g., "programming" -> "program" + "ming").
- **Input vs. Output Tokens**
    - **Input**: The user prompt and context provided to the model.
    - **Output**: The tokens generated by the model during inference.
    - **Autoregression**: Output tokens are iteratively fed back into the model as input to generate the subsequent token.
- **Chatbot**
    - A software wrapper around an LLM that maintains state and simulates conversation.

## V. Context Window and Memory Management
- **Context Window**
    - The fixed-size "buffer" of tokens the model can "see" at any one time (e.g., 128k, 200k, 1M tokens).
- **Transcript vs. Context**
    - **Transcript**: The literal history of prompts and responses (logical sequence).
    - **Context**: The mathematical state (activations) resulting from processing the transcript.
- **KV (Key-Value) Caching**
    - A technique to avoid re-calculating the mathematical state of previous tokens in a session.
    - **Block-based Storage**: Tokens are split into discrete blocks (e.g., 16 or 32 tokens) to prevent memory fragmentation in GPU RAM.
    - **Hashed Prefixes**: Each block is uniquely identified based on its content and all preceding tokens.
    - **Causal Attention Constraint**: The "Stack" logic; to understand token $N$, the model requires the state of tokens $1$ through $N-1$.
    - **Eviction and Re-computation**:
        - If a middle block is removed from the cache, all subsequent blocks become mathematically invalid.
        - A "break in the chain" forces a **Replay** (re-processing) from the point of the break.
- **Sliding Window / Eviction**
    - When the transcript exceeds the context window, the oldest tokens are "evicted" (dropped) to make room for new ones.
- **Compacting**
    - Summarizing or "garbage collecting" a long transcript to fit more relevant information into the fixed context window.

## VI. Agentic Paradigms and Advanced Tooling
- **Retrieval-Augmented Generation (RAG)**
    - Providing the model with relevant external documents (retrieved via search/vector DB) inside its context window to reduce **Hallucination** and provide up-to-date facts.
- **In-context Learning**
    - The model's ability to "learn" a task or adapt its style based purely on examples provided in the current prompt (without weight updates).
- **Tools and MCP**
    - **Tools**: Functions the model can "call" (e.g., search_web, run_code).
    - **Model Context Protocol (MCP)**: An open standard for connecting AI models to external data sources and tools.
- **Agents**
    - Systems where an LLM is given a goal and iteratively uses tools and reasoning to achieve it.
    - **ReAct Pattern (Reasoning + Acting)**:
        - **Thought**: The model plans its next step.
        - **Action**: The model calls a tool (e.g., searches a file).
        - **Observation**: The model reads the tool's output and updates its "Thought" for the next turn.
    - **Sub-agents / Orchestration**: A lead agent delegating specific tasks to specialized "workers."
    - **Agentic Loops**: Recursive processes where the model evaluates its own output and adjusts (Plan -> Act -> Validate).
- **Skills**
    - Encapsulated sets of instructions or tool-use patterns that extend an agent's capabilities.
- **Memory**
    - Long-term storage of user preferences or facts across different sessions (e.g., via a database or `save_memory` tools).
